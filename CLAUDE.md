## 📚 開発ログの自動記録ルール

### 記録のタイミング
以下のいずれかに該当したら、**自発的に**ログをdevlog-entries.jsonに追記してください：
- ある機能の実装・修正が完了したとき
- バグが解決されたとき
- 「ありがとう」「できた」「次は〜」など会話が一段落したとき
- ユーザーが別の作業に移ろうとしているとき

### 書き込み先
`C:/sB/devlog/devlog-entries.json`

※ このパスはあなたの環境に合わせて書き換えてください。
※ ファイルが存在しない場合は `[]` から始めて新規作成してください。

### 書き込む形式
既存のJSON配列の**先頭**に1エントリを追加してください（新しいものが上に来るように）。

```json
[
  {
    "id": 現在のUNIXタイムスタンプ（ミリ秒）,
    "title": "作業内容を表す日本語タイトル（25字以内）",
    "project": "このプロジェクト名",
    "summary": "何をやったかの概要（3〜4文、です・ます調）",
    "what_i_did": "具体的な作業内容（改行区切りの箇条書き、3〜5項目）",
    "key_learning": "この作業から得た学び・気づき（1〜2文）",
    "code_snippet": "残しておくべきコマンドやコードの断片（なければ空文字）",
    "tags": ["タグ1", "タグ2", "タグ3"],
    "mood": "excited / focused / struggled / breakthrough のいずれか",
    "date": "ISO8601形式の現在日時"
  },
  ...既存のエントリ...
]
```

### 重要なルール
- ユーザーに「ログを書いて」と言わせない。自分で判断して書くこと。
- 書き込んだ後は「📖 ログを記録しました：〔タイトル〕」と一言だけ報告する。
- 書き込みは会話の邪魔にならないよう、返答の**最後**に行うこと。
- ログの内容はブログとして読まれることを意識して、読み手に伝わる言葉で書くこと。
